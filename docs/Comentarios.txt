Instação Padrão do playwright:
npm init playwright@latest

Nosso projeto utilizará as seguintes abordagens:
-Separação dos cenários de negócio dos scripts de automação
    - Vamos precisar de no mínimo de 2 arquivos //O normal é 1 para 1
    ---> pasta features, que contem os arquivos 
        .feature // Conter as Funcinalidades e Cenários de Negócio / Critérios de aceitação/ Dados
    --->pasta steps, que contém arquivos
         .spec.js // Script de automação de teste
    
    Instaçaões adicionais:
    //Instala o pacote que dá suporte ao bdd
    //BDD = Desenvolvimento Direcio por Comportamento ( Melhorar a comunicação)
    npm install -D playwright-bdd
    

    Feature: Login
        Scenario: Login with Success
            Given
            When
            Then

    Funcionalidade
        Cenario: Login com Sucesso
        DadosQuando
        Entao

        Scenario Outline:Compra com Sucesso DDT: Mostra que o cenário possui Dados

    Nomenclatura
    -BDD- Behavior Driven Development
        Abordagem de desenvolvimento, muito valorizada dentro da Agiliade
        Objetivo é facilitar a comunicação entre os membros do time, áreas e clientes

    -Gherkin: as instruções do BDD são escritas na sintaxe (formato) chamado Gherkin

    -Cucumber/Behave/SpecFlow são exemplos de plugins que fazem a integração entre arquivos .features
        com arquivos .spec.js
    
    -Page Objects / Page Factory: sseparar o mapeamento dos elementos de interface gráfica (páginas, telas) do código de automação
    --> Pages com mapeamento de elementos
    --> Steps com os scripts de automação
    --> Features com os cenários de teste


Em um projeto simples, cada teste é um arquivo .spec.js diferente
Em um projeto com Gherkin, cada teste é composto por um arquivo .feature + um arquivo steps.js
